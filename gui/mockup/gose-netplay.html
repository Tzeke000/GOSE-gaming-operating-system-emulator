Relay servers are RetroArch's public MITM infrastructure — they relay traffic so NAT-blocked players can connect without port-forwarding. No relay = direct IP (fastest, but requires port-forwarding or LAN). Relay does not guarantee connectivity — server availability is RetroArch's, not GOSE's.
What's plumbing-verified vs needs-two-peers:
Verified: config read/write (nickname/port/relay written to batocera.conf); host + join launch commands built correctly (emulatorlauncher with correct -netplaymode/-netplayip flags); kill-by-PID works.
Not verifiable single-machine: the RetroArch frame-sync / rollback handshake — that requires a second peer to actually connect. "Waiting for client" is correct behavior, not a bug.
